# VAD and KG Reactive Linker

## Objective
Make VAD config, intelligence filters, and KG node updates fully reactive but non-looping, with one-way data flow and proper store subscriptions.

## Implementation
1. **Live Graph Additive Updates**: During `gemini_intelligence` events, the live graph is updated additively. Existing nodes are preserved, and new nodes are appended:
   ```javascript
   const existingNodeIds = new Set(graphNodes.map((n) => n.id));
   const newNodes = graphUpdate.nodes.filter((n) => !existingNodeIds.has(n.id));
   const mergedNodes = [...graphNodes, ...newNodes];
   ```
2. **Post-Processing Graph Override Prevention**: Prevented the heavy `extractKnowledgeGraph` function from running *while* recording is active. It now only runs during post-processing (`!isRecording`), ensuring that live graph data generated by the event listeners isn't wiped out or duplicated.
3. **Partial Transcript De-duplication**: Implemented exact chunk ID matching for removing partial Whisper transcripts, ensuring late-arriving chunks don't accidentally wipe out valid data.
4. **Insights Debouncing**: Added a `typingTimer` with a 2-second debounce to prevent UI flickering during rapid intelligence extraction cycles.
